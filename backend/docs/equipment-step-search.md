# Ступенчатый поиск оборудования Brunswick

Ниже — быстрые сценарии для ручной проверки API поиска по иерархии категорий.

## Корневой уровень (бренд/верхние категории)
1. **Запрос**: `GET /api/equipment/categories?brand=Brunswick&level=1`
2. **Ожидаемый ответ**: массив с корневым элементом `Brunswick` (`parentId = null`).

## Переход к категориям бренда
1. **Запрос**: `GET /api/equipment/categories?brand=Brunswick&parentId=1000`
2. **Ожидаемый ответ**: дочерние категории бренда (например, `Пинсеттеры`, `Скоринг-системы`, `Системы возврата шара`, `Натричные машины`, `Электроника` и др.) отсортированные по `sortOrder`.

## Просмотр моделей внутри категории
1. **Шаг 1**: получить `id` нужной категории (например, `Пинсеттеры` → `1100`).
2. **Шаг 2**: запросить потомков: `GET /api/equipment/categories?brand=Brunswick&parentId=1100`
3. **Ожидаемый ответ**: модели пинсеттеров `Boost ST (стринг-кегли)`, `GS NXT`, `GS-X`, `GS-98`, `GS-96`, `GS-92`.

## Защита от некорректных переходов
1. **Запрос без parentId для глубокого уровня**: `GET /api/equipment/categories?level=3`
   - **Ожидаемо**: ошибка 400 с требованием указать `parentId`.
2. **Запрос с несуществующим parentId**: `GET /api/equipment/categories?brand=Brunswick&parentId=999999`
   - **Ожидаемо**: пустой список.
